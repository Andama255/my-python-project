Overview

This page demonstrates core HTML5 concepts, focusing on creating semantic, accessible, and SEO-optimized web pages. You'll see practical examples of structure, accessibility enhancements, and metadata for search engines.

Semantic Structure with HTML5

HTML5 introduces semantic elements like <header>, <nav>, <main>, <section>, <article>, and <footer> which give meaning to web content and improve both human and machine understanding.

Example Semantic Tags

  • <header>: Page or section header
  • <nav>: Navigation links
  • <main>: Primary page content
  • <section> / <article>: Grouping related content
  • <footer>: Footer or copyright info

Accessibility Best Practices

Accessibility ensures your site is usable for everyone. Use semantic tags for landmarks, add aria attributes for assistive tech, and always use alt text for images (if present).

  • Use landmarks (<main>, <nav>, <header>, <footer>)
  • Logical heading order: <h1> → <h2> → <h3>
  • Use aria-label and aria-labelledby as needed
  • Text content should be clear and high-contrast (when using CSS)

SEO: Search Engine Optimization

SEO helps search engines understand your content. Some basics:

  • Use one <h1> per page (should match page topic)
  • Add a <meta name="description"> tag in the <head>
  • Use descriptive headings and plain language
  • Organize content with semantic sections and clear hierarchy

Conclusion

Using semantic HTML5, basic accessibility, and beginner SEO, you can build user-friendly and discoverable web pages. Follow these fundamentals to grow your web development skills!
